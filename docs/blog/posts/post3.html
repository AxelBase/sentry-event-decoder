Works Offline in 3 Seconds

From Event ID to Full Trace in 3 Seconds (Even Offline)

Published: November 17, 2025

You’re on a plane. Or in a data center with no Wi-Fi. Or on-call at 2 AM and Sentry is slow. An alert just fired. You need the trace now.

That’s why Sentry Event Decoder was built as a full Progressive Web App (PWA).

Install Once, Use Forever — Offline

After your first visit, the entire tool is cached locally using a Service Worker. All libraries (vis-network, html2canvas, FileSaver) are bundled and cached. You can close the tab, turn off Wi-Fi, and reopen — it just works.

Manual JSON Paste = Zero Dependencies

Since public event JSON fetching isn’t possible (Sentry requires auth), we made the best possible UX: just copy the JSON from Sentry and paste it. No upload. No server. Instant render.

Real Offline Incident Story

Last month, I was on a flight when a critical alert came in. I opened the installed PWA, pasted the JSON I’d saved earlier, and within 10 seconds knew a third-party API was timing out. Fixed on the ground. No internet needed.

Performance Matters

The app loads in less than 800ms on repeat visits. The trace renders in less than 200ms. Total time from paste to insight: less than 3 seconds.

FAQ

How do I install it?

Chrome/Firefox will show an install button. Or click “Add to Home Screen” on mobile.

Does it work on mobile?

Yes — fully responsive. Paste JSON on your phone, pinch to zoom the trace.

What if I clear my cache?

Just reload once with internet — it re-caches everything automatically.

In debugging, speed isn’t a feature — it’s oxygen. This tool gives you air when you’re drowning in alerts.
