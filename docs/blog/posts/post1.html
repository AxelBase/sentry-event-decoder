Introducing Sentry Event Decoder – Analyze Traces Instantly

Published: November 17, 2025

Every developer has been there: a production alert fires, you click into Sentry, wait for the page to load, navigate to the trace… and then wait again. What if you could skip all that and see the full performance trace in under 3 seconds — even offline?

That’s exactly what Sentry Event Decoder does.

Built as a 100% client-side, open-source web app, this tool lets you paste any Sentry event JSON (or even just the event ID) and instantly get a full interactive trace visualization with:

  • Red/Yellow/Green duration indicators (greater than 1s red, 500ms–1s yellow, less than 500ms green)
  • Dashed red lines showing error propagation from root to failing span
  • Zoom, pan, and collapse nodes powered by vis-network
  • One-click export to PNG or JSON
  • Works completely offline after first visit (PWA)

Why I Built This

I was tired of waiting for Sentry’s UI during on-call incidents. When checkout slowed down for 40% of users, I needed to see the trace now — not after three loading screens. So I built a tool that works instantly, anywhere.

No Login. No Rate Limits. No Data Sent.

Everything runs in your browser. Paste the JSON from Sentry, and the trace renders immediately. No backend. No tracking. No data leaves your machine.

Real-World Use Cases

Whether you're triaging a P95 regression, hunting down an N+1 query, or sharing a trace in Slack — this tool gets you answers fast.

FAQ

Can I use this with private Sentry events?

Yes! Just copy the full event JSON from Sentry and paste it — the tool works 100% offline.

Is it really free forever?

Yes. Open source, no limits, no account needed. MIT licensed.

Can I deploy my own version?

Absolutely. One-click deploy to GitHub Pages. Your own private analyzer in 60 seconds.
